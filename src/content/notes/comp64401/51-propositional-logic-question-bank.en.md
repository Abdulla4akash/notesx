# COMP64401 Chapter 1 — Propositional Logic Worked Question Bank

Source sheet read: `COMP64401 Chapter 1 - Propositional Logic.mht`.

This bank drills only the task types that appear in the sheet: propositional syntax, valuations, truth tables, satisfiability, validity, equivalence, entailment, reductions to SAT, NNF rewriting, tableau satisfiability, correctness ideas, and KR knowledge-base checks.

---

## Task types identified from the sheet

1. **Syntax classification:** decide whether an expression is a propositional formula under the primitive language `{¬, ∧, ∨}`.
2. **Semantic evaluation:** compute the truth value of a compound formula under a valuation.
3. **Truth-table reasoning:** build/read truth tables and classify formulae as satisfiable, unsatisfiable, valid, or not valid.
4. **Equivalence and entailment checks:** compare truth columns or search for entailment counterexamples.
5. **Syntactic sugar expansion:** replace `⇒` and `⇔` using only primitive operators.
6. **Reasoning-task reductions:** use satisfiability to decide validity and entailment.
7. **NNF conversion:** push negations inward using De Morgan rules and double-negation elimination.
8. **Tableau SAT:** initialise `𝒮`, apply `∧`-rules and `∨`-rules, detect clashes, and read the yes/no result.
9. **Tableau correctness vocabulary:** identify termination, soundness, completeness, and decision-procedure claims.
10. **KR checks:** represent a KB as a conjunction, check consistency/non-vacuity, desired entailments, and undesired-property unsatisfiability.

## Q1. Syntax classification: primitive PL formula or not?

For each expression, decide whether it is a propositional formula under the primitive syntax from the sheet.

1. `(p ∧ q) ∨ ¬r`
2. `p ⇒ q`
3. `∀x. P(x) ∧ Q(x)`
4. `¬(p ∨ (q ∧ r))`

### Solution

**Step 1: Recall the primitive syntax.**

A formula is built from variables in `𝒫` using only:

- variables such as `p, q, r`;
- negation `¬φ`;
- conjunction `φ ∧ ψ`;
- disjunction `φ ∨ ψ`;
- parentheses for grouping.

**Step 2: Check expression 1.**

`(p ∧ q) ∨ ¬r` uses only variables, `∧`, `∨`, `¬`, and parentheses.

So it **is** a propositional formula.

**Step 3: Check expression 2.**

`p ⇒ q` uses implication `⇒`.

In the primitive syntax of the sheet, `⇒` is not primitive. It can later be introduced as syntactic sugar, but before expansion it is **not yet** a primitive propositional formula.

So under primitive syntax, it is **not** a formula.

**Step 4: Check expression 3.**

`∀x. P(x) ∧ Q(x)` uses a quantifier `∀x` and predicates `P(x), Q(x)`.

These belong to predicate/first-order style syntax, not this basic propositional syntax.

So it is **not** a propositional formula.

**Step 5: Check expression 4.**

`¬(p ∨ (q ∧ r))` uses only variables, `¬`, `∨`, `∧`, and parentheses.

So it **is** a propositional formula.

**Final answers:**

| Expression | Primitive PL formula? | Reason |
|---|---:|---|
| `(p ∧ q) ∨ ¬r` | Yes | Generated by the syntax rules. |
| `p ⇒ q` | No, not yet | `⇒` is syntactic sugar, not primitive. |
| `∀x. P(x) ∧ Q(x)` | No | Uses quantifiers and predicates. |
| `¬(p ∨ (q ∧ r))` | Yes | Generated by the syntax rules. |

## Q2. One-step semantic evaluation

Let `v(p)=0`, `v(q)=1`, and `v(r)=1`. Evaluate:

`¬(p ∨ q) ∧ r`.

### Solution

**Step 1: Write the formula.**

`φ = ¬(p ∨ q) ∧ r`

**Step 2: Evaluate the inner disjunction.**

`v(p ∨ q) = max(v(p), v(q))`

`= max(0, 1)`

`= 1`

**Step 3: Evaluate the negation.**

`v(¬(p ∨ q)) = 1 - v(p ∨ q)`

`= 1 - 1`

`= 0`

**Step 4: Evaluate the final conjunction.**

`v(¬(p ∨ q) ∧ r) = v(¬(p ∨ q)) · v(r)`

`= 0 · 1`

`= 0`

**Final answer:**

The formula is **false** under `v`, because `v(φ)=0`.

## Q4. Read satisfiability and validity from a final truth-table column

A formula is satisfiable iff at least one valuation makes it true.

So we ask: is there at least one `1` in the final column?

There are two `1`s.

Therefore `φ` is **satisfiable**.

**Step 2: Test validity.**

A formula is valid iff every valuation makes it true.

So we ask: is every final-column entry `1`?

No. There are `0`s in rows 2 and 4.

Therefore `φ` is **not valid**.

**Final answer:**

`φ` is **satisfiable but not valid**.

## Q5. Expand syntactic sugar

Rewrite the formula below using only `¬`, `∧`, and `∨`:

`p ⇔ q`

### Solution

**Step 1: Expand biconditional.**

By definition:

`p ⇔ q := (p ⇒ q) ∧ (q ⇒ p)`

So:

`p ⇔ q`

becomes:

`(p ⇒ q) ∧ (q ⇒ p)`

**Step 2: Expand each implication.**

By definition:

`φ ⇒ ψ := ¬φ ∨ ψ`

So:

`p ⇒ q` becomes `¬p ∨ q`.

`q ⇒ p` becomes `¬q ∨ p`.

**Step 3: Substitute back.**

`(p ⇒ q) ∧ (q ⇒ p)`

becomes:

`(¬p ∨ q) ∧ (¬q ∨ p)`

**Final answer:**

`p ⇔ q` expands to:

`(¬p ∨ q) ∧ (¬q ∨ p)`

## Q6. Is this formula in NNF?

Decide whether each formula is in Negation Normal Form.

1. `¬p ∧ (q ∨ ¬r)`
2. `¬(p ∨ q) ∧ r`
3. `(¬p ∨ q) ∧ (r ∨ ¬s)`

### Solution

**Step 1: Recall the NNF discriminator.**

A formula is in NNF iff every negation occurs directly in front of a propositional variable.

So `¬p` is allowed, but `¬(p ∨ q)` is not.

**Step 2: Check formula 1.**

`¬p ∧ (q ∨ ¬r)`

Negations occur only in `¬p` and `¬r`.

Both are directly in front of variables.

So formula 1 **is in NNF**.

**Step 3: Check formula 2.**

`¬(p ∨ q) ∧ r`

The negation applies to the compound formula `(p ∨ q)`.

So formula 2 **is not in NNF**.

**Step 4: Check formula 3.**

`(¬p ∨ q) ∧ (r ∨ ¬s)`

Negations occur only in `¬p` and `¬s`.

Both are directly in front of variables.

So formula 3 **is in NNF**.

**Final answers:**

| Formula | In NNF? | Reason |
|---|---:|---|
| `¬p ∧ (q ∨ ¬r)` | Yes | Negations only before variables. |
| `¬(p ∨ q) ∧ r` | No | Negation applies to a compound formula. |
| `(¬p ∨ q) ∧ (r ∨ ¬s)` | Yes | Negations only before variables. |

## Q7. Apply one NNF rewrite rule

Rewrite the following one step closer to NNF:

`¬(p ∧ q)`

### Solution

**Step 1: Identify the pattern.**

The formula has the form:

`¬(φ₁ ∧ φ₂)`

where:

`φ₁ = p`

`φ₂ = q`

**Step 2: Apply the De Morgan NNF rule.**

The rule is:

`¬(φ₁ ∧ φ₂) ↝ ¬φ₁ ∨ ¬φ₂`

**Step 3: Substitute `p` and `q`.**

`¬(p ∧ q) ↝ ¬p ∨ ¬q`

**Final answer:**

`¬(p ∧ q)` rewrites to:

`¬p ∨ ¬q`

### Solution

The formula is:

`p ∧ q`

For a conjunction, both conjuncts must be true.

So if `φ₁ ∧ φ₂ ∈ S`, add both `φ₁` and `φ₂` to the same branch.

**Step 3: Apply the rule.**

Here:

`φ₁ = p`

`φ₂ = q`

So:

`S = {p ∧ q, r}`

becomes:

`S' = {p ∧ q, r, p, q}`

**Final answer:**

Apply the **`∧`-rule**. The branch becomes:

`{p ∧ q, r, p, q}`

No branching happens.

## Q9. Detect a clash

For each branch, say whether it contains a clash.

1. `S₁ = {p, ¬q, r}`
2. `S₂ = {p, ¬q, q}`
3. `S₃ = {¬p, q ∨ r, r}`

### Solution

**Step 1: Recall the clash condition.**

A branch contains a clash iff for some propositional variable `x`, both `x` and `¬x` occur in the same branch.

**Step 2: Check `S₁`.**

`S₁ = {p, ¬q, r}`

It contains `p`, `¬q`, and `r`.

It does not contain both `q` and `¬q`, and it does not contain both `p` and `¬p`.

So `S₁` is **clash-free**.

**Step 3: Check `S₂`.**

`S₂ = {p, ¬q, q}`

It contains both `q` and `¬q`.

So `S₂` **contains a clash**.

**Step 4: Check `S₃`.**

`S₃ = {¬p, q ∨ r, r}`

It contains `¬p`, but not `p`.

It contains `r`, but not `¬r`.

The formula `q ∨ r` is not itself a literal clash.

So `S₃` is **clash-free** at this point.

**Final answers:**

| Branch | Clash? | Reason |
|---|---:|---|
| `S₁` | No | No variable appears with its negation. |
| `S₂` | Yes | Contains both `q` and `¬q`. |
| `S₃` | No | No literal pair `x, ¬x`. |

## Q10. Running example: evaluate a formula under a valuation

Use the lecture’s running formula:

`φ = (p ∧ q) ∨ ¬(q ∨ r)`

Evaluate it under:

`v(p)=1`, `v(q)=1`, `v(r)=1`.

### Solution

**Step 1: Write the formula and valuation.**

`φ = (p ∧ q) ∨ ¬(q ∨ r)`

`v(p)=1`, `v(q)=1`, `v(r)=1`

**Step 2: Evaluate the inner disjunction `q ∨ r`.**

`v(q ∨ r) = max(v(q), v(r))`

`= max(1, 1)`

`= 1`

**Step 3: Evaluate the negation `¬(q ∨ r)`.**

`v(¬(q ∨ r)) = 1 - v(q ∨ r)`

`= 1 - 1`

`= 0`

**Step 4: Evaluate the conjunction `p ∧ q`.**

`v(p ∧ q) = v(p) · v(q)`

`= 1 · 1`

`= 1`

**Step 5: Evaluate the outer disjunction.**

`v((p ∧ q) ∨ ¬(q ∨ r)) = max(v(p ∧ q), v(¬(q ∨ r)))`

`= max(1, 0)`

`= 1`

**Final answer:**

The formula is **true** under this valuation.

`v(φ)=1`.

## Q12. Classify formulae as satisfiable, unsatisfiable, valid, or not valid

Classify each formula.

1. `p ∨ ¬p`
2. `p ∧ ¬p`
3. `p ∨ q`

### Solution

### Formula 1: `p ∨ ¬p`

**Step 1: Check possible values of `p`.**

If `p=1`, then `p ∨ ¬p = 1 ∨ 0 = 1`.

If `p=0`, then `p ∨ ¬p = 0 ∨ 1 = 1`.

**Step 2: Classify.**

The formula is true under every valuation.

So it is **valid**.

Every valid formula is also satisfiable.

So it is **satisfiable and valid**.

### Formula 2: `p ∧ ¬p`

**Step 1: Check possible values of `p`.**

If `p=1`, then `p ∧ ¬p = 1 ∧ 0 = 0`.

If `p=0`, then `p ∧ ¬p = 0 ∧ 1 = 0`.

**Step 2: Classify.**

The formula is false under every valuation.

So it is **unsatisfiable**.

Since it is not true under every valuation, it is **not valid**.

### Formula 3: `p ∨ q`

**Step 1: Find one satisfying valuation.**

Let `p=1`, `q=0`.

Then `p ∨ q = 1 ∨ 0 = 1`.

So the formula is satisfiable.

**Step 2: Find one counterexample to validity.**

Let `p=0`, `q=0`.

Then `p ∨ q = 0 ∨ 0 = 0`.

So the formula is not valid.

**Final answers:**

| Formula | Satisfiable? | Valid? | Classification |
|---|---:|---:|---|
| `p ∨ ¬p` | Yes | Yes | Valid tautology. |
| `p ∧ ¬p` | No | No | Unsatisfiable contradiction. |
| `p ∨ q` | Yes | No | Satisfiable but not valid. |

## Q14. Test entailment by searching for a counterexample

Does the following semantic entailment hold?

`p ∨ q → p`

Here `→` is the lecture’s entailment notation, not object-language implication.

### Solution

**Step 1: Recall the entailment test.**

`φ → ψ` holds iff there is no valuation where:

- `φ = 1`; and
- `ψ = 0`.

So for `p ∨ q → p`, search for a valuation where:

- `p ∨ q = 1`; and
- `p = 0`.

**Step 2: Try to make the conclusion false.**

The conclusion is `p`.

To make it false, set:

`p = 0`

**Step 3: Try to make the premise true anyway.**

The premise is `p ∨ q`.

If `p=0`, make `q=1`.

Then:

`p ∨ q = 0 ∨ 1 = 1`

**Step 4: Check the counterexample.**

Under `v(p)=0`, `v(q)=1`:

- premise `p ∨ q` is true;
- conclusion `p` is false.

This is exactly the forbidden entailment counterexample.

**Final answer:**

No. `p ∨ q` does **not** entail `p`.

Counterexample:

`v(p)=0`, `v(q)=1`.

## Q16. Decide whether a system-description formula is meaningful

A system is represented by formula `α`. For each final truth-table column below, classify the system as broken, vacuous, or meaningful.

1. `0, 0, 0, 0`
2. `1, 1, 1, 1`
3. `1, 0, 1, 0`

### Solution

**Step 1: Recall the KR discriminator.**

A useful system description should be:

- **satisfiable**, so it is not self-contradictory;
- **not valid**, so it actually constrains possible states.

So:

- all `0`s means unsatisfiable → broken;
- all `1`s means valid → vacuous;
- mixed `1`s and `0`s means satisfiable but not valid → meaningful/constraining.

**Step 2: Classify column 1.**

`0, 0, 0, 0`

There is no satisfying valuation.

So the system is **broken / inconsistent**.

**Step 3: Classify column 2.**

`1, 1, 1, 1`

Every valuation satisfies the formula.

So the formula is valid and the system is **vacuous**.

It constrains nothing.

**Step 4: Classify column 3.**

`1, 0, 1, 0`

There is at least one `1`, so it is satisfiable.

There is at least one `0`, so it is not valid.

So the system is **meaningful / constraining**.

**Final answers:**

| Final column | Classification |
|---|---|
| `0, 0, 0, 0` | Broken / inconsistent. |
| `1, 1, 1, 1` | Vacuous / unconstraining. |
| `1, 0, 1, 0` | Meaningful / constraining. |

## Q18. Build a KB conjunction and sanity-check it

Use the formulae from Q17:

1. `R ⇒ W`
2. `R ⇒ U`
3. `(R ∧ S) ⇒ B`

Let:

`KB = (R ⇒ W) ∧ (R ⇒ U) ∧ ((R ∧ S) ⇒ B)`

Show that this KB is satisfiable and not valid.

### Solution

**Step 1: Recall the sanity checks.**

A useful KB should be:

- satisfiable: at least one valuation makes it true;
- not valid: at least one valuation makes it false.

**Step 2: Find a valuation that makes the KB true.**

Choose:

`R=0`, `W=0`, `U=0`, `S=0`, `B=0`

Now evaluate each implication.

`R ⇒ W` is true because the antecedent `R` is false.

`R ⇒ U` is true because the antecedent `R` is false.

`(R ∧ S) ⇒ B` is true because `R ∧ S = 0 ∧ 0 = 0`, so the antecedent is false.

Therefore all conjuncts are true.

So `KB=1` under this valuation.

Hence the KB is **satisfiable**.

**Step 3: Find a valuation that makes the KB false.**

To make a conjunction false, make one conjunct false.

Make `R ⇒ U` false by setting:

`R=1`, `U=0`

For example choose:

`R=1`, `W=1`, `U=0`, `S=0`, `B=0`

Then:

`R ⇒ U` is false, because the antecedent is true and the consequent is false.

So the whole conjunction `KB` is false.

Hence the KB is **not valid**.

**Final answer:**

The KB is **satisfiable but not valid**, so it is not broken and not vacuous.

## Q19. Running NNF example from the sheet

Convert the following formula to NNF:

`¬(¬(p ∧ ¬(q ∧ r)) ∨ q)`

### Solution

**Step 1: Identify the outer pattern.**

The formula is:

`¬(¬(p ∧ ¬(q ∧ r)) ∨ q)`

The outermost form is:

`¬(A ∨ B)`

where:

`A = ¬(p ∧ ¬(q ∧ r))`

`B = q`

**Step 2: Apply De Morgan to the outer negated disjunction.**

`¬(A ∨ B) ↝ ¬A ∧ ¬B`

So:

`¬(¬(p ∧ ¬(q ∧ r)) ∨ q)`

rewrites to:

`¬¬(p ∧ ¬(q ∧ r)) ∧ ¬q`

**Step 3: Eliminate double negation.**

`¬¬(p ∧ ¬(q ∧ r)) ↝ p ∧ ¬(q ∧ r)`

So the formula becomes:

`(p ∧ ¬(q ∧ r)) ∧ ¬q`

**Step 4: Push negation into `q ∧ r`.**

Use:

`¬(q ∧ r) ↝ ¬q ∨ ¬r`

So:

`(p ∧ ¬(q ∧ r)) ∧ ¬q`

becomes:

`(p ∧ (¬q ∨ ¬r)) ∧ ¬q`

**Step 5: Check NNF.**

Every negation is now directly in front of a variable:

`¬q`, `¬r`, `¬q`

So the result is in NNF.

**Final answer:**

`¬(¬(p ∧ ¬(q ∧ r)) ∨ q)`

converts to:

`(p ∧ (¬q ∨ ¬r)) ∧ ¬q`

`p ∧ (¬q ∧ ¬r)`

All negations are directly in front of variables.

`¬(p ⇒ (q ∨ r))`

## Q21. Running tableau example from the sheet

Run the set-of-sets tableau algorithm on:

`φ = (p ∧ (¬q ∨ ¬r)) ∧ ¬s`

This formula is already in NNF.

### Solution

The tableau starts with:

`𝒮 = {{φ}}`

So:

`𝒮 = {{(p ∧ (¬q ∨ ¬r)) ∧ ¬s}}`

The formula has the form:

`A ∧ B`

where:

`A = p ∧ (¬q ∨ ¬r)`

`B = ¬s`

The `∧`-rule adds both conjuncts to the same branch.

So:

`𝒮 = {{φ, p ∧ (¬q ∨ ¬r), ¬s}}`

Now decompose:

`p ∧ (¬q ∨ ¬r)`

This adds:

`p`

and:

`¬q ∨ ¬r`

`𝒮 = {{φ, p ∧ (¬q ∨ ¬r), ¬s, p, ¬q ∨ ¬r}}`

Now the branch contains:

`¬q ∨ ¬r`

One branch adds `¬q`.

The other branch adds `¬r`.

`𝒮 = {S₁, S₂}`

`S₁ = {φ, p ∧ (¬q ∨ ¬r), ¬s, p, ¬q ∨ ¬r, ¬q}`

**Step 6: Apply the return condition.**

The tableau returns “yes” iff at least one final branch is clash-free.

Here both final branches are clash-free.

**Final answer:**

The tableau returns **yes**.

`(p ∧ (¬q ∨ ¬r)) ∧ ¬s`

## Q24. Decide validity using a SAT solver

Use the reduction from the sheet to decide whether this formula is valid:

`φ = p ∨ ¬p`

Assume your SAT solver can answer satisfiability questions.

### Solution

**Step 1: Recall the reduction.**

A formula `φ` is valid iff `¬φ` is not satisfiable.

So to decide validity of `φ`, ask whether `¬φ` is satisfiable.

**Step 2: Negate the formula.**

`φ = p ∨ ¬p`

So:

`¬φ = ¬(p ∨ ¬p)`

**Step 3: Convert the negation to a simpler form.**

Using De Morgan:

`¬(p ∨ ¬p) ↝ ¬p ∧ ¬¬p`

Using double-negation elimination:

`¬p ∧ p`

**Step 4: Ask whether `¬p ∧ p` is satisfiable.**

No valuation can make both `p` and `¬p` true.

So `¬p ∧ p` is unsatisfiable.

**Step 5: Convert the SAT answer back.**

Since `¬φ` is not satisfiable, `φ` is valid.

**Final answer:**

`p ∨ ¬p` is **valid**.

## Q25. Decide entailment using a SAT solver

### Solution

**Step 1: Recall the reduction.**

`φ → ψ` holds iff:

`φ ∧ ¬ψ`

is **not satisfiable**.

**Step 2: Identify premise and conclusion.**

Premise:

`φ = p ∧ q`

Conclusion:

`ψ = p`

**Step 3: Build the counterexample formula.**

`φ ∧ ¬ψ = (p ∧ q) ∧ ¬p`

**Step 4: Ask whether the counterexample formula is satisfiable.**

`(p ∧ q) ∧ ¬p`

requires both:

`p`

and:

`¬p`

This is impossible.

So the counterexample formula is unsatisfiable.

**Step 5: Convert the SAT answer back.**

Because no counterexample is satisfiable, the entailment holds.

**Final answer:**

Yes. `p ∧ q → p` holds.

`¬(¬BZ ∨ ¬SO)`

`¬¬BZ ∧ ¬¬SO`

`BZ ∧ SO`

## Q28. Vacuous entailment from an inconsistent premise

Does this semantic entailment hold?

`p ∧ ¬p → q`

### Solution

**Step 1: Recall the entailment counterexample test.**

`φ → ψ` fails only if there is a valuation where:

- `φ = 1`; and
- `ψ = 0`.

Here:

`φ = p ∧ ¬p`

`ψ = q`

So we search for a valuation where:

`p ∧ ¬p = 1`

and:

`q = 0`

**Step 2: Check whether the premise can be true.**

`p ∧ ¬p` is never true.

If `p=1`, then `¬p=0`.

If `p=0`, then `¬p=1`, but `p=0`.

So `p ∧ ¬p` is always `0`.

**Step 3: Decide whether a counterexample exists.**

A counterexample needs the premise to be true.

But the premise is never true.

So no counterexample exists.

**Final answer:**

Yes, the entailment holds **vacuously**.

This is why, in KR, checking satisfiability of a KB is important before trusting its entailments. An inconsistent KB can entail formulae in an unhelpful way.

## Q34. Truth table vs tableau: why both methods agree but scale differently

Consider:

The formula is:

`(p ∨ q) ∧ ¬p`

`p=0`

`q=1`

`¬p = 1`

So:

`(p ∨ q) ∧ ¬p = 1 ∧ 1 = 1`

**Step 1: Initialise.**

`𝒮 = {{{(p ∨ q) ∧ ¬p}}}`

Call the input formula `φ`.

`𝒮 = {{φ}}`

**Step 2: Apply the `∧`-rule.**

Add both conjuncts:

`p ∨ q`

and:

`¬p`

So:

`𝒮 = {{φ, p ∨ q, ¬p}}`

Split on `p ∨ q`:

`S₁ = {φ, p ∨ q, ¬p, p}`

## Q35. Desired vs undesired property checks

A KB is represented by formula `KB`. You have two properties:

Which checks should you run?

### Solution

`KB → D`

is unsatisfiable.

is unsatisfiable.

If it is unsatisfiable, the bad situation is ruled out.

For desired properties, you check that the KB **guarantees** the property.

For undesired properties, you check that the KB **excludes** the property.

**Final answer:**

|---|---|---|
| Desired `D` | `KB → D`, equivalently `KB ∧ ¬D` unsatisfiable | Entailment holds. |

## Q36. Why “show all entailments” is not a good KB-tool output

A KB tool computes that `KB` entails `p ∨ ¬p`, `q ∨ ¬q`, `(p ∨ ¬p) ∧ (q ∨ ¬q)`, and many more tautological formulae. Why is this not useful, and what should the tool focus on instead?

### Solution

**Step 1: Notice that tautologies are always entailed.**

A tautology is valid, meaning it is true under every valuation.

So every KB entails every tautology, regardless of the KB’s actual content.

For example:

`p ∨ ¬p`

is valid.

Therefore `KB → (p ∨ ¬p)` holds for any `KB`.

**Step 2: Notice the infinite-output problem.**

You can generate endlessly many tautological consequences:

`p ∨ ¬p`

`q ∨ ¬q`

`(p ∨ ¬p) ∧ (q ∨ ¬q)`

and so on.

So “all entailments” is not a practical or meaningful output.

**Step 3: Identify what should be filtered out.**

The tool should avoid flooding the user with:

- tautologies;
- formulae already present in the KB;
- equivalent restatements;
- huge lists of uninteresting consequences.

**Step 4: Identify useful KB checks.**

The tool should focus on:

- satisfiability of the KB;
- non-validity of the KB;
- user-specified desired properties;
- user-specified undesired properties;
- counterexamples that help repair the KB.

**Final answer:**

Showing all entailments is not useful because infinitely many trivial entailments exist. A useful KB tool should focus on consistency, non-vacuity, desired-property entailments, undesired-property unsatisfiability, and helpful counterexamples.

Because the branch is clash-free, it does not contain both `x` and `¬x` for any variable `x`.

So the valuation is well-defined.

It never forces both `v(x)=1` and `v(x)=0`.

**Step 4: Use the tableau-rule invariant.**

The tableau rules preserve satisfiability along a clash-free final branch: the formulae on that branch can be made true together by the valuation extracted from the literals.

**Step 5: Use the fact that the input formula remains on the branch.**

The original input formula `φ` is still in `S`.

So the extracted valuation makes `φ` true.

Therefore `φ` is satisfiable.

**Final answer:**

A clash-free final branch gives a consistent valuation, and because `φ` remains on that branch, that valuation satisfies `φ`. This is the core soundness idea.

## Q40. Choose the fastest reasoning shortcut for the running formula

For the running formula:

`φ = (p ∧ q) ∨ ¬(q ∨ r)`

A valuation has `v(p)=1` and `v(q)=1`, but `v(r)` is unknown. Can you already decide `v(φ)`?

### Solution

**Step 1: Look for a decisive subformula.**

The formula is:

`(p ∧ q) ∨ ¬(q ∨ r)`

The left disjunct is:

`p ∧ q`

**Step 2: Evaluate the left disjunct using known values.**

Given:

`v(p)=1`

`v(q)=1`

So:

`v(p ∧ q) = 1 · 1 = 1`

**Step 3: Use the outer disjunction.**

The whole formula is:

`(p ∧ q) ∨ ¬(q ∨ r)`

A disjunction is true if at least one side is true.

The left side is already true.

So the whole formula is true regardless of the right side.

**Step 4: Explain why `r` does not matter here.**

The value of `r` can change `¬(q ∨ r)`, but it cannot change the fact that the left disjunct is already true.

**Final answer:**

Yes. `v(φ)=1` no matter what `v(r)` is.

This is a shortcut, but the full semantic method would still evaluate subformulae systematically if needed.
