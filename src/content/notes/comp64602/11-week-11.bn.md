## 1.1 Verification বনাম validation

Slides দুটি সম্পর্কিত কিন্তু আলাদা প্রশ্ন আলাদা করে দেখায়:

- **Validation:** “Did we build the right system?”
  - অর্থাৎ system-টি intended real-world need বা user requirement-এর সঙ্গে মেলে কি না।
- **Verification:** “Did we build the system right?”
  - অর্থাৎ implementation-টি formal specification satisfy করে কি না।

### মূল ধারণা — verification

**Intuition:** Formal requirement-এর বিরুদ্ধে correctness পরীক্ষা করা।

**Slides-এ formalism:** একক formal definition হিসেবে দেওয়া নেই; তবে পরে model checking-কে এমনভাবে উপস্থাপন করা হয়েছে যে প্রতিটি possible execution sequence কোনো formula satisfy করে কি না তা পরীক্ষা করা হয়।

### মূল ধারণা — validation

**Intuition:** নির্বাচিত requirements/system সত্যিই সঠিক কি না তা পরীক্ষা করা।

**Slides-এ formalism:** কোনো formal definition দেওয়া নেই।

## 1.2 Testing, model checking, এবং theorem proving

Visual comparison তিনটি approach দেখায়।

### Testing

Testing নির্বাচিত execution-গুলোর ওপর behaviour পরীক্ষা করে।

- দেখানো প্রশ্ন: “শেষে কি \(\phi\) true?”
- এটি path/sample-based: শুধু tested execution পরীক্ষা করা হয়।

### Model checking

Model checking একটি model থেকে generated সব execution-এর ওপর properties পরীক্ষা করে।

- দেখানো example property: “যদি \(\phi\) true হয়, তবে eventually \(\psi\) true হবে।”
- Model checking একটি manually chosen test path-এর বদলে transition system explore করে।

### Theorem proving

Theorem proving symbolic reasoning করে।

- দেখানো example property: “যদি শুরুতে \(\phi\) true হয়, তবে শেষে \(\psi\) true হবে।”
- Diagram-এ \(\phi, \theta, \pi, \eta, \psi\) formula-সহ proof-like structure দেখানো হয়েছে।

## 1.4 System models: transition systems

Slides একটি transition system define করে এভাবে:

\[
\langle S, \to, \Pi, \pi, s_0\rangle
\]

যেখানে:

- \(S\): states-এর set।
- \(\to\): transition relation।
- \(\Pi\): atomic propositions-এর set।
- \(\pi : \Pi \to 2^S\): valuation function।
- \(s_0\): initial state।

[UNCLEAR / অস্পষ্ট] Slide-এর text-এ বলা আছে “Where M is a set of states,” কিন্তু tuple-এ \(S\) ব্যবহার করা হয়েছে। সম্ভবত এটি typo; intended component হলো \(S\), অর্থাৎ states-এর set।

### মূল ধারণা — transition system

**Intuition:** একটি graph-like model, যেখানে system কীভাবে state থেকে state-এ যেতে পারে তা দেখানো হয়, এবং কোন state-এ কোন proposition true তা label করা থাকে।

**Slides থেকে formal definition:**

\[
\langle S, \to, \Pi, \pi, s_0\rangle
\]

উপরের components-সহ।

## 1.5 Model checking: concept

Slides model checking এভাবে describe করে:

- একটি program-কে transition system হিসেবে model করা।
- সেই transition system ব্যবহার করে execution sequences generate করা।
- ধরা যাক \(\Sigma\) হলো finite set of execution sequences, যা কোনো program বা component-এর সব possible executions represent করে।
- তারপর check করা হয়:

\[
\forall \sigma \in \Sigma . \langle \sigma, s_0\rangle \models \phi
\]

- Model checking fail করলে, এটি failed sequence-টি **counter-example** হিসেবে return করতে পারে।
- Notation: states-এর sequence \(\sigma\)-এর \(i\)-th position-এ যেসব propositions true, তাদের set হলো \(\pi(i)\)।

### মূল ধারণা — model checking

**Intuition:** একটি model-এর প্রতিটি possible execution exhaustively check করা যে সেটি formal property satisfy করে কি না।

**Slides থেকে formal version:**

\[
\forall \sigma \in \Sigma . \langle \sigma, s_0\rangle \models \phi
\]

## 1.6 States-এর sequence-এর ওপর LTL semantics

**Exam flag:** Slide title স্পষ্টভাবে বলে LTL semantics **not examinable**।

Examples বোঝার জন্য semantics এখনও useful।

### Atomic propositions

\[
\langle \sigma, s_i\rangle \models p
\quad \text{iff} \quad
p \in \pi(i)
\]

যেখানে \(p \in \Pi\)।

### Negation

\[
\langle \sigma, s_i\rangle \models \neg \phi
\]

iff এটি সত্য নয় যে:

\[
\langle \sigma, s_i\rangle \models \phi
\]

### Conjunction

\[
\langle \sigma, s_i\rangle \models \phi \land \psi
\]

iff:

\[
\langle \sigma, s_i\rangle \models \phi
\]

এবং:

\[
\langle \sigma, s_i\rangle \models \psi
\]

### Disjunction

\[
\langle \sigma, s_i\rangle \models \phi \lor \psi
\]

iff:

\[
\langle \sigma, s_i\rangle \models \phi
\]

অথবা:

\[
\langle \sigma, s_i\rangle \models \psi
\]

### Next

\[
\langle \sigma, s_i\rangle \models \bigcirc \phi
\quad \text{iff} \quad
\langle \sigma, s_{i+1}\rangle \models \phi
\]

### Eventually

\[
\langle \sigma, s_i\rangle \models \Diamond \phi
\]

iff এমন একটি \(j\) আছে যাতে:

\[
j \geq i
\]

এবং:

\[
\langle \sigma, s_j\rangle \models \phi
\]

### Always

\[
\langle \sigma, s_i\rangle \models \Box \phi
\]

iff সব \(j\)-এর জন্য, যদি \(j \geq i\), তবে:

\[
\langle \sigma, s_j\rangle \models \phi
\]

### Until

\[
\langle \sigma, s_i\rangle \models \phi \, U \, \psi
\]

iff এমন একটি \(j\) আছে যাতে:

\[
j \geq i
\]

এবং:

\[
\langle \sigma, s_j\rangle \models \psi
\]

এবং সব \(k\)-এর জন্য, যদি:

\[
j > k \geq i
\]

তাহলে:

\[
\langle \sigma, s_k\rangle \models \phi
\]

Slide note করে যে \(\Box\) এবং \(\Diamond\) কে \(U\) এবং \(\neg\) ব্যবহার করে express করা যায়।

## 1.7 Worked example 1: একটি true LTL property check করা

দেখানো sequence হলো:

\[
c \to b \to a \to \cdots \to e
\]

Check করার property:

\[
\langle \sigma, s_0\rangle \models c \land \bigcirc b \land \bigcirc\bigcirc a
\]

Conjunction অনুযায়ী, এটি তিনটি check-এ reduce করে:

1. Check:

\[
\langle \sigma, s_0\rangle \models c
\]

যার মানে:

\[
c \in \pi(0)
\]

2. Check:

\[
\langle \sigma, s_0\rangle \models \bigcirc b
\]

যার মানে:

\[
b \in \pi(1)
\]

3. Check:

\[
\langle \sigma, s_0\rangle \models \bigcirc\bigcirc a
\]

যার মানে:

\[
a \in \pi(2)
\]

যেহেতু sequence-এ position 0-এ \(c\), position 1-এ \(b\), এবং position 2-এ \(a\) আছে, formula satisfied।

---

## 1.8 Worked example 2: একই sequence, ভিন্ন property

একই sequence ব্যবহার করা হয়েছে:

\[
c \to b \to a \to \cdots \to e
\]

Check করার property:

\[
\langle \sigma, s_0\rangle \models c \land \bigcirc c \land \bigcirc\bigcirc \neg a
\]

Conjunction অনুযায়ী এটি reduce করে:

1. Check:

\[
\langle \sigma, s_0\rangle \models c
\]

যার মানে:

\[
c \in \pi(0)
\]

এটি true।

2. Check:

\[
\langle \sigma, s_0\rangle \models \bigcirc c
\]

যার মানে:

\[
c \in \pi(1)
\]

এটি true নয়, কারণ position 1-এ \(b\) আছে, \(c\) নয়।

3. Check:

\[
\langle \sigma, s_0\rangle \models \bigcirc\bigcirc \neg a
\]

যার মানে:

\[
a \notin \pi(2)
\]

এটি true নয়, কারণ position 2-এ \(a\) আছে।

তাই পুরো conjunction false।

# 2. CTL, ATL, এবং অন্যান্য logics

উৎস: *COMP64602: CTL, ATL and Other Logics*

## 2.1 কেন CTL/CTL*?

LTL-এ formulas-কে model-এর **সব path**-এ true হতে হয়। কিন্তু কখনও কখনও আমরা শুধু **কিছু path**-এ truth নিয়ে কথা বলতে চাই।

**Computational Tree Logic / CTL\*** path quantifiers introduce করে:

- \(A\): সব paths-এর জন্য।
- \(E\): একটি path আছে।

Temporal operator আছে এমন property-তে temporal operator-এর আগে কোথাও একটি path quantifier থাকতে হবে।

যে কোনো LTL formula \(\phi\) CTL* এ translate হয়:

\[
A\phi
\]

অর্থাৎ LTL-style “সব path-এ true” interpretation CTL* এ explicit হয়।

## 2.2 CTL* semantics

**Exam flag:** Slide title স্পষ্টভাবে বলে **CTL\* semantics are not examinable**।

Slides বলে CTL* LTL semantics-কে “in the obvious way” extend করে।

### Atomic proposition

\[
\langle M, s_i\rangle \models p
\quad \text{iff} \quad
p \in \pi(i)
\]

\(p \in \Pi\)-এর জন্য।

### Until

\[
\langle M, \sigma\rangle \models \phi U \psi
\]

iff এমন একটি \(j\) আছে যাতে:

\[
s_j \in \sigma
\]

\[
j \geq 0
\]

\[
\langle M, s_j\rangle \models \psi
\]

এবং সব \(k\)-এর জন্য, যদি:

\[
s_k \in \sigma
\]

এবং:

\[
j > k \geq 0
\]

তাহলে:

\[
\langle M, s_k\rangle \models \phi
\]

### Universal path quantifier

\[
\langle M, s_i\rangle \models A\phi
\]

iff:

\[
\langle M, \sigma\rangle \models \phi
\]

\(s_i\) থেকে শুরু হওয়া সব path \(\sigma\)-এর জন্য।

### Existential path quantifier

\[
\langle M, s_i\rangle \models E\phi
\]

iff:

\[
\langle M, \sigma\rangle \models \phi
\]

\(s_i\) থেকে শুরু হওয়া কোনো একটি path \(\sigma\)-এর জন্য।

## 2.3 CTL*, এর fragment হিসেবে CTL

CTL হলো CTL* এর এমন একটি fragment যেখানে প্রতিটি temporal operator, যেমন:

\[
\Box
\]

অথবা:

\[
\Diamond
\]

এর ঠিক আগে path quantifier থাকতে হবে।

### CTL formula example

\[
A \Box E \bigcirc red
\]

Slide-এ দেওয়া meaning: সব path-এ সবসময় এমন হয় যে একটি path আছে যেখানে next step-এ \(red\) true।

এটি CTL formula, কারণ temporal operators-গুলো সঙ্গে সঙ্গে path quantifiers-এর সঙ্গে paired।

### CTL formula নয়

\[
A \Box \Diamond red
\]

Slide-এ দেওয়া meaning: সব path-এ সবসময় এমন হয় যে eventually \(red\) true হবে।

এটি **CTL নয়**, কারণ \(\Diamond\)-এর ঠিক আগে path quantifier নেই।

Slides বলে CTL-এর computational properties CTL* থেকে ভালো, তাই automated reasoning দরকার হলে CTL preferable।

## 2.4 CTL formulae-এর examples

### “It is always the case that it is possible for the system to stop next.”

\[
A \Box E \bigcirc stop
\]

### “Before the robot starts moving it always sounds a warning.”

\[
\neg move \land A \Box (A \bigcirc(\neg move) \lor warning)
\]

[UNCLEAR / অস্পষ্ট] Slide formula-টি সরাসরি দেয়। Derivation expand করে না, এবং \(warning\)-এর placement next-time operator-এর relative কোথায়/কেন তা explain করে না।

### “All requests are eventually acknowledged.”

\[
A \Box(req \Rightarrow A \Diamond ack)
\]

### “It is possible for the system to achieve its goal.”

\[
E \Diamond goal
\]

### “It is always possible to move to a fail-safe state.”

\[
A \Box E \Diamond fail\text{-}safe
\]

## 2.5 Alternating Time Temporal Logic: ATL*

Multi-agent systems-এ আমরা প্রায়ই জানতে চাই groups of agents একসঙ্গে কাজ করে কিছু achieve করতে পারে কি না।

ATL* CTL-কে generalise করে, যেখানে path quantifier \(A\) এবং \(E\)-এর বদলে **cooperation modalities** ব্যবহার হয়:

\[
\langle\langle A\rangle\rangle
\]

যেখানে \(A\) হলো agents-এর team।

Formula:

\[
\langle\langle A\rangle\rangle \phi
\]

express করে যে agent team \(A\)-এর power আছে \(\phi\) true করানোর।

CTL* এর মতোই, temporal operator থাকা property-তে তার আগে cooperation modality থাকতে হবে। ATL-এ cooperation modality-টি temporal operator-এর ঠিক আগে থাকতে হবে।

### মূল ধারণা — cooperation modality

**Intuition:** “এই coalition-এর এমন strategy আছে যা outcome-টি force করতে পারে”—এ কথা বলার উপায়।

**Slides থেকে formalism:**

\[
\langle\langle A\rangle\rangle \phi
\]

মানে team \(A\) \(\phi\) ensure করতে পারে।

## 2.6 ATL* high-level semantics

ATL* semantics concurrent agent actions দিয়ে explain করা হয়েছে।

যে কোনো state-এ:

- Agents যে সব actions নিতে পারে সেগুলো consider করা হয়।
- একটি transition function আছে:

\[
\tau
\]

যা নেয়:

\[
s
\]

এবং actions-এর একটি tuple:

\[
(\alpha_1,\ldots,\alpha_n)
\]

এবং next state return করে।

এখানে:

\[
\alpha_i
\]

হলো agent \(a_i\)-এর নেওয়া action।

আরও একটি action function আছে:

\[
d
\]

যা প্রতিটি state-এ প্রতিটি agent-এর available actions define করে।

নিচেরটি valid হতে হলে:

\[
\tau(s,\alpha_1,\ldots,\alpha_n)
\]

প্রতিটি chosen action available হতে হবে:

\[
\alpha_i \in d(a_i,s)
\]

এই structure-কে **concurrent game structure** বলা হয়।

## 2.7 Worked example: দুই robot এবং একটি carriage

Slide দেখায়:

- একটি carriage।
- Positions:
  - \(pos_0\)
  - \(pos_1\)
  - \(pos_2\)
- States:
  - \(S0\)

# 3. Modular Interpreted Systems

উৎস: *COMP64602: Modular Interpreted Systems*

## 3.1 Model checking: basic idea

Lecture model-checking method পুনরায় বলে:

1. System-কে কোনো ধরনের transition system হিসেবে represent করা, যেমন concurrent game structure।
2. Property-এর negation-কে আরেকটি transition system হিসেবে represent করা।
3. Negated-property transition system system model-এর মধ্য দিয়ে “bad paths” represent করে।
4. দুটিকে combine করা।
5. Combination empty কি না check করা।
6. Combination empty হলে system model-এ কোনো bad path possible নয়।

### মূল ধারণা — bad path

**Intuition:** Model-এর মধ্য দিয়ে এমন path যা check করা property violate করে।

**Slides থেকে formalism:** Property-এর negation এমন transition system হিসেবে represented হয় যার paths হলো bad paths।

## 3.4 Modular Interpreted Systems: core idea

## 3.5 Agent representation

প্রতিটি individual agent একটি transition system হিসেবে represented হয়:

### Perfect information

## 4.1 Gwendolen reasoning cycle

### Stage E

New perception এবং messages নেওয়া।

### Stage F

Final MCAPL slide robot/planning scenario-এর typical properties দেয়।

### Planning succeeds

\[
PlanningSucceeds(Pos):
\]

\[
\Box(
D_{ag1}query(get\_close\_to(Pos, P))
\Rightarrow
\Diamond B_{ag1}have\_plan(Pos, plan)
)
\]

Meaning: যখনই agent \(ag1\) \(Pos\)-এর কাছে যেতে query perform করে, eventually agent \(ag1\) believe করে যে তার \(Pos\)-এর জন্য একটি plan আছে।

### Plan execution succeeds

\[
PlanExecutionSucceeds(Pos):
\]

\[
\Box(
D_{ag1}perf(execute(plan))
\Rightarrow
\Diamond B_{ag1}in\_position(Pos)
)
\]

Meaning: যখনই agent \(ag1\) plan execute করে, eventually agent \(ag1\) believe করে যে সে position \(Pos\)-এ আছে।

### Combined property

\[
(
PlanningSucceeds(middle)
\land
PlanExecutionSucceeds(middle)
)
\Rightarrow
\Diamond B_{ag1}maintaing(middle)
\lor
B_{ag1}broken(x)
\lor
B_{ag1}broken(y)
\lor
B_{ag1}broken(z)
\]

[UNCLEAR / অস্পষ্ট] Slide-এ `maintaing(middle)` লেখা আছে, যা `maintaining(middle)`-এর typo মনে হয়, কিন্তু recording/source code দেখে check না করা পর্যন্ত notes-এ slide-এর spelling preserve করা উচিত। এছাড়া slide final disjunction-এ operator precedence clarify করে না।

# 5. Cross-lecture connections

## 5.1 LTL → CTL/CTL*

LTL lecture states-এর sequences-এর ওপর formulas দিয়ে শুরু করে। CTL/ATL lecture তারপর এটি extend করে বলে যে LTL properties সব paths-এর ওপর interpreted, আর CTL* \(A\) এবং \(E\) ব্যবহার করে path quantification explicit করে। যে কোনো LTL formula \(\phi\) CTL* এ \(A\phi\) হিসেবে translate হতে পারে।

## 5.2 CTL/ATL → MIS

CTL/ATL lecture states, joint actions, এবং transition functions ব্যবহার করে concurrent game structures introduce করে। MIS একই multi-agent motivation ব্যবহার করে, কিন্তু system-কে modularly represent করার ওপর focus করে: individual agents, তাদের actions, এবং information-এর মাধ্যমে।

## 5.3 MIS → strategic reasoning

MIS প্রতিটি agent-এর exact implementation না জেনেও possible behaviour নিয়ে reasoning support করে। ATL* strategies নিয়ে reason করার logic, যার মধ্যে perfect/imperfect information এবং perfect/imperfect recall ভিত্তিক variations আছে।

## 5.4 Operational semantics → program model checking

MCAPL lecture abstract transition systems থেকে actual agent programming languages-এ shift করে। Gwendolen programs-এর operational semantics আছে, এবং program instructions program state-এ transitions cause করে। তাই program execution model-এর ওপর সরাসরি model checking apply করা natural।

## 5.5 সাধারণ temporal logic-এর বাইরে agent properties

Final lectures plain temporal properties-এর বাইরে যায় এবং properties নিয়ে কথা বলে যেখানে আছে:

- beliefs,
- goals,
- actions/deeds,
- commitments,
- agent strategies,
- protocols।

Examples:

\[
B(ag1, obstacle) \to \bigcirc G(ag1, stop)
\]

এবং:

\[
\Diamond B(ag2, win)
\]

এগুলো ordinary LTL/CTL থেকে richer, কারণ এগুলো agent-এর mental attitudes বা agent-specific events refer করে।
